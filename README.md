## System Architecture

This pipeline is broken down into four distinct quantitative engineering modules:

### 1. Data Engineering (Polars)
* Designed an asynchronous data ingestion pipeline capable of handling large-scale tick data.
* Compressed 37,000,000 raw Level 1 trade events into 18,500 volume-clocked bars to normalize the statistical distribution of market volatility.

### 2. Alpha Generation (Scikit-Learn)
* Engineered lagged Order Flow Imbalance (OFI) features.
* Trained a Ridge Regression model to predict micro-price movements over short horizons, establishing a directional edge (56.20% out-of-sample hit rate).

### 3. Execution Engine (Avellaneda-Stoikov)
* Implemented a customized Avellaneda-Stoikov continuous-time market-making model.
* Built a grid-search optimizer to dynamically adjust risk aversion, spread scaling, and machine learning alpha weighting to eliminate adverse selection during momentum regimes.

### 4. Autonomous Reporting (CrewAI + Local LLM)
* Integrated a CrewAI agent utilizing a locally hosted Llama 3.2 model.
* The agent ingests raw mathematical backtest outputs and autonomously drafts professional, desk-ready investment memos.

## Technology Stack

* Data Processing: Python, Polars, PyArrow
* Machine Learning: Scikit-Learn, NumPy
* AI Orchestration: CrewAI, Ollama (Llama 3.2)
